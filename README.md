# Техническая документация: Модуль преобразователя PT100 → RS485 (Modbus RTU)

**Версия:** 1.0 (Уточненная)
**Область применения:** Промышленный сбор данных о температуре, интеграция в системы автоматизации (в т.ч. KC868-A16).

## 1. Аппаратные характеристики

* **Интерфейс:** RS-485 (двухпроводной: A+, B-).
* **Вход:** PT100 (поддержка 2-х и 3-х проводных схем подключения).
* **Питание:** Уточните по маркировке на плате (обычно 5V-24V).
* **Трансивер:** 75P20A03 (или аналог).
* **Контроллер:** Эмуляция Modbus RTU Slave.

## 2. Карта регистров (Modbus Holding Registers)

Доступ осуществляется через стандартную функцию **FC03** (чтение) и **FC06** (запись).

| Адрес регистра (HEX) | Описание | Тип | Диапазон/Значение |
| --- | --- | --- | --- |
| **0x0100** | Текущая температура | Read-only | Значение * 0.1 °C (int16) |
| **0x0002** | Modbus Address (ID) | R/W | 1–247 (Flash) |
| **0x0003** | Baudrate Code | R/W | 0=1200, 1=2400, 2=4800, 3=9600 (Flash) |
| **0x0004** | Калибровочный Offset | R/W | Смещение в 0.1 °C (int16, Flash) |

*Примечание: Параметры, помеченные как (Flash), сохраняются после отключения питания.*

## 3. Протокол коммуникации

* **Параметры UART:** 8-N-1 (8 бит данных, нет четности, 1 стоп-бит).
* **Скорость по умолчанию:** 9600 baud.
* **Адрес по умолчанию:** 1.

## 4. Методика настройки и калибровки

### Изменение адреса устройства

Для работы нескольких датчиков на одной шине RS-485 необходимо изменить адрес каждого модуля по очереди:

1. Подключите модуль к контроллеру (один модуль на шине).
2. Запишите новое значение (например, `2`) в регистр `0x0002` через функцию `FC06`.
3. Модуль примет адрес немедленно, перезагрузка не требуется.

### Изменение скорости обмена

Если требуется иная скорость, запишите соответствующий код в регистр `0x0003`:

* `0` — 1200 baud
* `1` — 2400 baud
* `2` — 4800 baud
* `3` — 9600 baud

### Калибровка температуры

Если показания датчика отклоняются от эталонных:

1. Считайте текущую температуру из `0x0100`.
2. Рассчитайте разницу (Offset).
3. Запишите разницу в регистр `0x0004` (например, для коррекции +1.5 °C запишите `15`).

## 5. Важные предостережения

* **Broadcast Address:** Избегайте записи `0` в регистр `0x0002` (Адрес). Это может привести к потере управления над модулем из-за перевода его в широковещательный режим.
* **Зависание:** Не записывайте в регистры скорости значения, отличные от кодов 0–4, во избежание сбоев работы UART контроллера.
* **Целостность данных:** Поскольку параметры `0x0002`, `0x0003` и `0x0004` пишутся в энергонезависимую память, не следует производить запись в них в цикле `loop()` вашего контроллера (это быстро исчерпает ресурс записи Flash-памяти чипа). Выполняйте запись только при необходимости конфигурации.
